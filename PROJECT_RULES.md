# Правила работы над проектом

## Обязательные правила для всех агентов:

1. **НЕ коммитить и НЕ пушить** изменения пока пользователь явно не попросит об этом
2. **Перед пушем ОБЯЗАТЕЛЬНО проверять тесты** - запускать `npm test` и убедиться что все тесты проходят
3. Создавать тесты для нового функционала перед коммитом
4. Исправлять ошибки линтера перед сохранением файлов

## Рабочий процесс:

1. Делать изменения в коде
2. Создавать/обновлять тесты
3. Запускать тесты: `npm test`
4. Исправлять ошибки
5. **ЖДАТЬ явного разрешения пользователя** перед коммитом и пушем
6. Перед коммитом еще раз проверить тесты: `npm test`
7. Только после проверки тестов делать коммит и пуш

## Важно:

- Всегда проверять что тесты проходят перед коммитом
- Не коммитить без явного разрешения пользователя
- Создавать тесты для нового функционала
